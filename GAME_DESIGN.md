# GAME_DEXSIGN

## 개요

현재 프로젝트는 `폐허 서울의 3일 생존`을 다루는 텍스트 RPG Stage 1 MVP다.  
플레이어는 전쟁으로 무너진 서울의 생활권 안에서 먹을 것, 물, 돈, 잠자리를 확보하며 3일을 버틴다.  
3일차 밤이 지나면 하늘에 게이트가 열리는 사건이 발생하고, Stage 1은 종료된다.

이 문서는 **현재 코드에 실제로 구현된 구조**를 기준으로 정리한 것이다.

## 핵심 플레이 판타지

- 영웅이 되기 전, 먼저 살아남아야 하는 사람의 이야기
- 거대한 운명보다 당장의 식사, 물, 휴식, 동선이 더 절실한 생존 일상
- 폐허 서울의 생활권을 자유롭게 돌며 루트를 찾고, 위험과 허기를 관리하는 탐험
- 3일차에 가까워질수록 도시 곳곳에서 설명할 수 없는 이상 징후가 짙어지는 긴장감

## 현재 플레이 루프

기본 루프는 아래 순서로 설계되어 있다.

1. 장소를 이동한다.
2. 해당 장소에서 탐색, 거래, 식사, 휴식, 정찰 중 하나를 고른다.
3. 포만감, 정신력, 체력, 소지품, 돈이 변한다.
4. 새로운 장소나 정보, 소문, 루트가 열린다.
5. 현실 시간이 흐르며 게임 내 시계가 자동으로 진행된다.
6. 시간이 흐르면서 포만감이 자동으로 줄어들고, 밤이 되면 거처에서 식사와 잠자리를 정리한다.

## 시간 시스템

현재 시간 시스템은 **행동 기반 시간 소모가 아니라 실시간 기반 자동 진행**이다.

- 실제 시간 `15분`이 게임 시간 `1일`
- 게임 시간은 `아침 -> 낮 -> 저녁 -> 밤 -> 새벽` 순환
- 상단 UI에는 `HH:MM` 형식의 게임 내 시계가 표시됨
- 시계 표시는 `10분 단위`로 끊어서 보임
- 내부 시간은 계속 흐르지만, 플레이어에게 보이는 시간은 `06:00`, `06:10`, `06:20`처럼 표시됨
- 하루가 넘어갈 때 자동으로 다음 날 아침 상태가 적용됨
- 현실 시간이 흐를수록 포만감도 자동으로 감소함

### 시간 관련 구현 포인트

- 내부 기준: `REAL_DAY_MS = 15분`
- 하루를 5개 시간대로 분할
- `worldElapsedMs`를 기준으로 `day`, `phaseIndex` 계산
- `setInterval`로 매초 시간 동기화
- 자동 허기 감소는 `AUTO_FULLNESS_TICK_MS = REAL_DAY_MS / 4`
- 장기 굶주림 시간 누적은 `STARVATION_TICK_MS = REAL_DAY_MS / 2`
- 시간대 변화나 날짜 변화가 발생하면 상태와 장면이 갱신됨

## 일차 전환 규칙

날짜가 바뀌면 자동으로 다음 규칙이 적용된다.

- 취침 전 포만감을 기준으로 소폭의 아침 포만감 보정
- 다음 날의 `끼니 확보`, `물 확보` 플래그 초기화
- 이전 날 허기 상태에 따라 장기 굶주림 누적치 반영
- 4일차에 진입하면 `gate_opening`으로 이동

### 포만감 감소 규칙

- 기본 감소는 시간이 흐르면서 자동으로 발생한다.
- 자동 감소는 `게임 하루당 약 4칸`을 목표로 한다.
- 즉, 실제 시간 기준 `약 3분 45초`마다 포만감이 `1` 줄어든다.
- 밤을 넘길 때는 추가로 큰 폭으로 깎지 않고, 포만감이 높은 상태에서만 소폭 보정한다.
- 목표 체감은 `하루에 2끼 정도` 챙기면 안정적으로 유지되는 수준이다.

## 스탯 구조

현재 메인 스탯은 3개다.

- `체력`
- `정신력`
- `포만감`

모든 스탯은 내부적으로 `0~10` 범위를 사용한다.

### 포만감 표시 규칙

포만감은 숫자를 직접 보여주지 않고 구간형 텍스트로 표시한다.

- `9~10`: 완전 배부름
- `7~8`: 배부름
- `4~6`: 평범
- `2~3`: 배고픔
- `1`: 매우 배고픔
- `0`: 기력 없음

## 생존 규칙

### 포만감

- 시간이 흐르면 포만감이 자동으로 감소한다.
- 일반적인 이동, 탐색, 노동, 정찰은 포만감을 직접 소모하지 않는다.
- 포만감은 기본적으로 `시간 경과 + 식사/회복`으로 관리된다.
- 아주 무리한 특수 행동만 예외적으로 추가 포만감 소모를 가질 수 있다.
- 음식과 물은 포만감을 회복한다.
- 포만감이 `0`이 되더라도 즉시 체력이 깎이지는 않는다.

### 장기 굶주림

현재 구조에는 `starvationLevel`이라는 내부 누적치가 있다.

- 포만감이 `0`인 상태가 시간상으로 오래 유지되면 누적치가 쌓인다.
- 포만감이 `0`인 상태에서 아주 무리한 특수 행동을 하면 누적이 더 빨라질 수 있다.
- 누적치가 기준 이상이 되면 체력이 깎인다.
- 식사나 회복 행동은 이 누적치를 줄인다.

즉, 시스템 의도는 다음과 같다.

- `포만감`: 당장 배가 고픈가
- `장기 굶주림`: 며칠째 제대로 못 먹어서 몸이 망가지고 있는가

### 정신력

- 정신력은 거친 현실과 이상 징후를 버티는 수치다.
- 지하철역, 하천변, 통제선 인근 같은 장소의 특정 행동에서 감소한다.
- 낮아지면 장면 설명 문구가 더 불안하게 바뀐다.
- `0`이 되면 게임 오버다.

### 체력

- 상처, 무리한 행동, 장기 굶주림 누적 등에서 감소한다.
- 진통제나 수면, 일부 안정 행동으로 회복할 수 있다.

## 행동 비용 구조

행동은 `ACTION_COSTS` 기반으로 정의된다.  
각 행동 타입은 포만감, 정신력, 체력, 장기 굶주림 누적에 영향을 준다.

현재 주요 행동 타입은 다음과 같다.

- `move`: 이동, 기본 포만감 소모 없음
- `light`: 가벼운 행동, 주로 포만감 소모 없음
- `search`: 수색, 기본 포만감 소모 없음
- `work`: 노동/보조, 기본 포만감 소모 없음
- `scout`: 정찰, 정신력 소모 중심
- `anomaly`: 이상 징후 확인, 포만감 소모 + 큰 정신력 소모
- `snack`, `cannedMeal`, `meal`, `drink`, `medicine`: 회복 행동

이 구조 덕분에 새 이벤트를 만들 때 공통 비용 체계를 재사용할 수 있다.

## 스킬 구조

현재 플레이 시작 시 3가지 생존 방식 중 하나를 선택한다.

- `눈썰미`
- `버티기`
- `흥정`

### 스킬 의미

- `눈썰미`: 단서, 숨은 경로, 낙서, 이상 징후를 먼저 읽는 감각
- `버티기`: 수색이나 위험 행동에서 체력 손실을 덜 받음
- `흥정`: 거래 비용 절감, 사람과의 교환 이득 증가

## 아이템 구조

현재 아이템은 생존 중심으로 구성되어 있다.

- `비상식량`
- `캔 음식`
- `따뜻한 식사`
- `물병`
- `진통제`
- `교환용 잡화`
- `배식권`

### 아이템 역할

- 음식류: 포만감 회복, 장기 굶주림 완화
- 물병: 포만감 소량 회복 + 정신력 소량 회복
- 진통제: 체력 회복
- 교환용 잡화: 거래용 재료
- 배식권: 급식소에서 무료 식사에 사용

## 장소 구조

현재 구현된 주요 장소는 아래와 같다.

- `임시 거처`
- `편의점 잔해`
- `급식소`
- `작은 상가 골목`
- `지하철역`
- `폐마트`
- `하천변`
- `통제선 인근`

### 장소별 역할

- `임시 거처`: 안전 거점, 휴식, 우선순위 정리, 잠자기
- `편의점 잔해`: 초기 식량/물/잔돈 확보
- `급식소`: 한 끼 식사, 물 확보, 배식 보조
- `작은 상가 골목`: 거래, 소문, 우회로 해금
- `지하철역`: 소문, 이상 징후, 통제선 방향 단서
- `폐마트`: 추가 식량, 약품, 현금, 교환품 확보
- `하천변`: 물, 거래, 소문, 이상 징후
- `통제선 인근`: 최종 이상 징후 확인, Stage 1 종료 조건

## 퀘스트 구조

퀘스트는 영웅 서사가 아니라 생존 우선순위를 보여주는 역할이다.

- `오늘 끼니 마련`
- `물 확보`
- `생활권 확장`
- `이상 징후 목격`
- `3일 생존`

`currentQuestHint()`는 현재 상태를 기준으로 플레이어에게 다음 우선순위를 제안한다.

예시:

- 1일차 초반: 식량과 물 확보 우선
- 2일차: 지하철역과 하천변에서 소문 수집
- 3일차: 이상 징후 확인
- 이상 징후 확인 후: 통제선 인근 정찰

## 현재 스토리 구조

### 오프닝

플레이어는 폐허가 된 서울의 임시 거처에서 눈을 뜬다.  
살기 위해 먹을 것과 돈을 구해야 하며, 어떤 방식으로 살아남아 왔는지 선택한다.

### 1일차

- 아침 첫 선택: 남은 식량을 먹을지, 아낄지, 밖에서 구할지 결정
- 핵심 목표: 오늘 끼니와 물, 최소한의 생활 자원 확보
- 주요 활동 장소: 편의점 잔해, 급식소, 골목

### 2일차

- 생활권이 넓어지고 소문이 등장
- 지하철역, 하천변, 골목에서 이상한 이야기 수집
- 우회로와 추가 탐색 루트 해금

### 3일차

- 도시 전체 분위기가 불안하게 바뀜
- 지하철역과 하천변에서 이상 징후를 직접 확인
- 통제선 인근으로 이어지는 길 해금

### 종료

- 통제선 인근에서 하늘이 갈라지는 현상을 목격
- `gate_opening` 장면으로 이동
- `stage1_clear`에서 Stage 1 종료

## 현재 구현상 주의점

- 게임 시간은 이제 행동으로 흐르지 않고 실시간으로만 흐른다.
- 포만감은 행동 소모만으로 줄지 않고, 시간 경과만으로도 자동 감소한다.
- 일반 행동은 포만감을 직접 깎지 않는다.
- 현재 예외적인 포만감 행동 소모는 고강도 특수 행동 위주로만 남아 있다.
- 행동 메타 문구에는 시간 소모가 표시되지 않는다.
- 수면은 다음 날로 강제 점프시키는 기능이 아니라, 거처에서 한 번 사용할 수 있는 회복 행동에 가깝다.
- 날짜 전환은 현실 시간 경과에 의해 자동 발생한다.
- 상단 시계는 내부 시간을 계속 반영하지만, 화면에는 `10분 단위`로만 보인다.
- 하루 2끼 정도를 기본 식사 리듬으로 상정한 밸런스다.
